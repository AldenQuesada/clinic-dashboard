Trio de ferramentas para elevar o ClinicAI Dashboard ao nível "premium de verdade" — registrado em 2026-04-14 para trazer em sprint futuro.

O problema

O dashboard hoje funciona, mas UI tem micro-interações básicas, animações limitadas e componentes feitos à mão. Para dar a sensação "produto de SaaS top tier" (estilo Linear, Vercel, Arc) precisamos de três camadas trabalhando juntas.

As três camadas

Framer Motion

Biblioteca React para animações declarativas, gestos, layout transitions e orquestração de sequências.

21st.dev (Magic MCP)

Registry de componentes shadcn-style com MCP que gera UI a partir de linguagem natural.

UX/UI Pro Max

Skills de auditoria e planejamento (design-an-interface, web-interface-architect, ux-design-expert).

Framer Motion — o que dá pra fazer

  • Layout animations automáticas — prop layout anima mudanças de posição/tamanho sem esforço (reordenar kanban, expandir cards)
  • Shared element transitions — layoutId cria "hero transition" entre rotas (foto de paciente do grid pra tela cheia)
  • Gestos nativos — drag, hover, tap, pan com física realista
  • Scroll-driven — useScroll + useTransform para parallax, progress bars, reveal animations
  • Variants + stagger — cascata de animações em listas (KPIs aparecendo em sequência)
  • AnimatePresence — enter/exit suave em modais, toasts, rota
  • Spring physics — animação com inércia natural, não linear fake

Exemplo concreto pro Cashflow

<motion.div
  layout
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ type: "spring", stiffness: 300, damping: 30 }}
  whileHover={{ y: -4, boxShadow: "0 12px 40px rgba(201,169,97,0.2)" }}
>
  <KPICard valor={receita} />
</motion.div>

21st.dev — o que dá pra fazer

  • Magic MCP — descrevo em linguagem natural e ele gera componente Tailwind+React no padrão shadcn
  • Biblioteca de componentes premium — hero sections, pricing tables, bento grids, navbars animados
  • Blocos inspirados em Aceternity/Magic UI — meteor effects, spotlight cards, border beam, animated gradients
  • Busca visual — encontra componentes prontos por screenshot/descrição
Setup necessário: claude mcp add magic npx -y @21st-dev/magic@latest (precisa de API key em 21st.dev)

UX/UI Pro Max — o que dá pra fazer

  • design-an-interface — gera múltiplas variantes radicalmente diferentes da mesma tela via sub-agents paralelos
  • web-interface-architect — planeja component system, interaction flow, layout responsivo
  • ux-design-expert (Uma) — auditoria Nielsen, acessibilidade, hierarquia visual, contraste
  • copywriting skill — revisa e melhora copy de landing/features/CTAs

Estimativa de esforço

Setup inicial (1 dia)

  • Instalar Framer Motion no projeto
  • Configurar MCP do 21st.dev + API key
  • Criar motion-tokens.js com durações/easings padronizados
  • Definir 3-4 variants globais (fadeIn, slideUp, stagger, scale)

Por página (0.5 a 2 dias)

  • Audit UX (30 min)
  • Gerar variantes de layout (1h)
  • Componentes via 21st (1-3h)
  • Integrar Framer Motion (2-4h)
  • QA acessibilidade + ajustes (1h)

Riscos e cuidados

  • Performance — Framer Motion é pesado; evitar em listas >50 itens sem virtualização
  • Motion sickness — respeitar prefers-reduced-motion sempre
  • Consistência — criar design tokens de motion (duração, easing) para não virar "cada página anima diferente"
  • Custo 21st.dev — Magic MCP é pago; validar ROI antes de assinar
  • Bundle size — Framer Motion adiciona ~50KB gzipped; considerar motion/react (lazy)
